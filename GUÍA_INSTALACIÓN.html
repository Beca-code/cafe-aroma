Café Aroma

Guía Completa de Instalación y Configuración

Requisitos Previos

Asegúrate de tener lo siguiente antes de comenzar:

Navegador Web Moderno

Chrome, Firefox, Safari o Edge (versión reciente)

Cuenta de Google

Necesaria para crear un proyecto en Firebase

Editor de Código

Visual Studio Code (recomendado) u otro editor de texto

Conexión a Internet

Requerida para Firebase y APIs externas

Paso 1: Configurar Firebase

1

Crear Proyecto en Firebase

1. Ve a Firebase Console
2. Haz clic en "Crear proyecto"
3. Dale el nombre "cafe-aroma"
4. Desactiva "Google Analytics" (opcional)
5. Haz clic en "Crear proyecto"

2

Habilitar Autenticación

1. Ve a Authentication en el menú izquierdo
2. Haz clic en "Primeros pasos"
3. Selecciona "Email/Contraseña"
4. Habilita la opción
5. Haz clic en "Guardar"

3

Crear Base de Datos Firestore

1. Ve a Firestore Database
2. Haz clic en "Crear base de datos"
3. Selecciona "Modo de prueba" (para desarrollo inicial)
4. Elige una ubicación cercana
5. Haz clic en "Crear"

4

Obtener Credenciales de Firebase

1. Haz clic en el ícono de engranaje ajistes en la esquina superior izquierda
2. Selecciona "Configuración del proyecto"
3. Ve a la pestaña "General"
4. Desplázate hacia abajo hasta encontrar "Tus apps"
5. Haz clic en "Configuración" de la aplicación Web
6. Copia el objeto firebaseConfig

Paso 2: Crear Colecciones en Firestore

Importante: Necesitas crear 4 colecciones manualmente en Firestore.
1

Colección: usuarios

1. En Firestore, haz clic en "Iniciar colección"
2. Escribe usuarios como ID
3. Haz clic en "Siguiente"
4. Crea un documento de ejemplo (o déjalo vacío por ahora)
Campos que contendrá:

  • nombre (string)
  • email (string)
  • rol (string: "admin" o "usuario")
  • fechaRegistro (timestamp)
  • uid (string)
2

Colección: bitacora_acceso_correcto

Repite el proceso anterior con ID: bitacora_acceso_correcto
Campos que contendrá:

  • fecha_hora (timestamp)
  • usuario (string)
  • rol (string)
  • id_sesion (string)
  • ip (string)
3

Colección: bitacora_acceso_fallido

Repite el proceso anterior con ID: bitacora_acceso_fallido
Campos que contendrá:

  • fecha_hora (timestamp)
  • email_intentado (string)
  • motivo (string)
  • ip (string)
4

Colección: bitacora_cierre_sesion

Repite el proceso anterior con ID: bitacora_cierre_sesion
Campos que contendrá:

  • fecha_hora (timestamp)
  • usuario (string)
  • rol (string)
  • duracion_sesion (number)

Paso 3: Configurar Reglas de Firestore

Para propósitos de desarrollo, usa reglas permisivas:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /{document=**} { allow read, write: if request.auth != null; } } }
Para Producción: Usa reglas más restrictivas que validen roles.

1. Ve a Firestore Database
2. Haz clic en la pestaña "Reglas"
3. Reemplaza el contenido con las reglas anteriores
4. Haz clic en "Publicar"

Paso 4: Configurar el Proyecto Localmente

1

Abrir Proyecto en VS Code

1. Abre Visual Studio Code
2. Ve a Archivo → Abrir carpeta
3. Selecciona la carpeta cafe-aroma
4. El proyecto está listo

2

Editar Configuración de Firebase

1. Abre el archivo firebase.js
2. Busca la sección const firebaseConfig = {
3. Reemplaza los valores TU_* con tus credenciales de Firebase
4. Guarda el archivo (Ctrl+S)

const firebaseConfig = { apiKey: "TU_API_KEY", // → Reemplaza aquí authDomain: "TU_AUTH_DOMAIN", // → Reemplaza aquí projectId: "TU_PROJECT_ID", // → Reemplaza aquí // ... más campos };
3

Instalar Extensión Live Server (Opcional)

1. En VS Code, ve a Extensiones (Ctrl+Shift+X)
2. Busca "Live Server"
3. Instala la extensión de Ritwick Dey
4. Haz clic derecho en index.html y selecciona "Open with Live Server"

🧪 Paso 5: Realizar Pruebas

1

Crear Cuenta de Prueba

1. Abre index.html (login)
2. Haz clic en "Regístrate aquí"
3. Crea una cuenta con:
  - Email: usuario@example.com
  - Contraseña: Test1234!
4. Deberías ser redirigido a user.html

2

Crear Cuenta de Administrador

1. Vuelve al login
2. Haz clic en "Regístrate aquí"
3. Crea una cuenta con:
  - Email: admin@admin.cafearoma.com
  - Contraseña: Admin1234!
4. Deberías ser redirigido a admin.html

3

Verificar Bitácoras

1. Inicia sesión como administrador
2. Ve a la pestaña "Accesos Correctos"
3. Deberías ver los registros de login que realizaste
4. Intenta acceder con una contraseña incorrecta y verifica "Accesos Fallidos"

Paso 6: Desplegar en Netlify

1

Opción A: Git + Netlify (Recomendado)

1. Crea un repositorio en GitHub
2. Sube tu proyecto
3. Ve a Netlify
4. Haz clic en "New site from Git"
5. Selecciona tu repositorio
6. Configure:
  - Build command: (dejar en blanco)
  - Publish directory: .
7. Haz clic en "Deploy"

2

Opción B: Netlify CLI

Ejecuta en terminal:

npm install -g netlify-cli netlify login netlify deploy --prod --dir=.
3

Opción C: Arrastra y Suelta

1. Ve a Netlify
2. Arrastra la carpeta cafe-aroma al área de suelta
3. Tu sitio estará en vivo en segundos

🐛 Solución de Problemas

Error: "Firebase is not defined"
Solución: Verifica que los scripts de Firebase estén antes de firebase.js en los archivos HTML.
Error: "Las bitácoras no se guardan"
Solución: Verifica que las colecciones existan en Firestore y que hayas publicado las reglas.
Error: "Permission denied on Firestore"
Solución: Ve a Firestore → Reglas y asegúrate de que permiten lectura/escritura.
Error: "No puedo obtener la IP"
Solución: Este error es normal en redes corporativas. La función caerá a "IP no disponible".
